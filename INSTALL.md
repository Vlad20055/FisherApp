# Fisher — руководство по установке

## Требования

- Windows 10/11 (или другая ОС с .NET 9 SDK).
- **PostgreSQL** — основная БД приложения (схема из `Infrastructure/Scripts/`).
- **Redis** — кэш, сессии JWT, счётчик неудачных входов, блэклист после 3 ошибок, pub/sub инвалидации кэша.
- **MongoDB** — журнал событий и отчёты (TTL 30 дней на поле `createdAt`).

## Сборка

Откройте `Fisher.sln` в Visual Studio 2022 или выполните:

```bash
dotnet build Fisher.sln
```

Запуск консоли:

```bash
dotnet run --project UI/UI.csproj
```

## Настройка подключений

1. **PostgreSQL** — класс `Infrastructure/Options/DbOptions.cs`: свойство `ConnectionString` (по умолчанию `localhost`, БД `fisher`, пользователь `postgres`).

2. **Redis** — переменная окружения `FISHER_REDIS` (строка подключения StackExchange.Redis), иначе `localhost,abortConnect=false`.

3. **MongoDB** — переменная `FISHER_MONGO` (например `mongodb://localhost:27017`), иначе локальный инстанс.

4. **JWT** — переменная `JWT_SECRET`: может быть **любой длины** (внутри ключ нормализуется до 256 бит через SHA-256). Для прода задайте длинную случайную строку.

## Порядок запуска сервисов

1. Запустите PostgreSQL и примените скрипты создания БД/функций при необходимости.
2. Запустите **Redis** и **MongoDB** до старта приложения (иначе при старте будет ошибка подключения).
3. Запустите `UI`.

## Проверка

- Вход под существующим пользователем: после трёх неверных паролей имя блокируется в Redis на **10 минут** (сообщение об этом при следующей попытке).
- Администратор: пункт меню **9** — поиск логов, агрегаты MongoDB, экспорт JSON/CSV.
